### 2. Исправленная версия ТЗ

1. Пользователь может добавить в корзину от 1 до 10 единиц одного товара.
2. Пользователь может изменить количество каждого товара в корзине. Минимальное количество — 1. Для удаления товара из корзины используется отдельная кнопка «Удалить».
3. В корзине может находиться не более 5 различных товаров.
4. Суммарное количество всех товаров в корзине не может превышать 50 штук.
5. При попытке добавить товар, превышающий лимиты, система показывает соответствующее сообщение:
   - «Превышен лимит количества одного товара (макс. 10)»;
   - «Превышен лимит различных товаров в корзине (макс. 5)»;
   - «Превышен общий лимит корзины (макс. 50)».
6. Цена на товар в корзине соответствует текущей цене в каталоге. При изменении цены рядом с товаром отображается уведомление: «Цена изменилась».
7. На странице корзины отображается: список товаров с изображением и названием, их количество, цена за единицу, общая стоимость позиции и общая сумма корзины.
8. Если товара нет в наличии, рядом с ним отображается метка «Нет в наличии».
9. В корзине может отображаться реклама других товаров — не более 1 блока в нижней части страницы.
10. Реклама показывается каждый будний день с 8:00 до 10:00 и с 18:00 до 21:00 по московскому времени.
11. Для авторизованных пользователей с платной подпиской реклама не отображается.
12. Корзина сохраняется для авторизованного пользователя между сессиями.
13. Для неавторизованного пользователя корзина хранится локально и привязывается к аккаунту после входа.

### 3. Уточняющие вопросы продукт-менеджеру

1. Цена (п.7 vs п.13): Какая стратегия выбрана — фиксация цены при добавлении или динамическое обновление при изменении в каталоге? От этого зависит вся логика корзины.
2. Удаление: Какой механизм удаления товара — только через кнопку «Удалить» или также через обнуление количества?
3. Снятие с продажи: Как система должна реагировать, если товар из корзины стал недоступен (закончился, снят с производства)?
4. Промокоды и скидки: Должна ли корзина поддерживать промокоды, скидки, купоны?
5. Пропущенный п.12: Это опечатка нумерации или было утерянное требование?
6. Неавторизованные пользователи: Как долго хранить корзину для неавторизованного пользователя?
7. Синхронизация: Нужна ли синхронизация корзины между устройствами?

### Как это работает

**1. Регистрация устройства**

После установки приложение получает push-токен (FCM для Android / APNs для iOS) и отправляет его на сервер:

```
POST /api/v1/push-tokens
```

```json
{
  "user_id": 12345,
  "platform": "ios",
  "push_token": "fj23jf8sdfk23..."
}
```

Notification Service сохраняет токен в базе.

**2. Возникает событие**

Примеры:

- Cart Service (товар 24 часа в корзине):
  ```json
  { "event_type": "cart_abandoned", "user_id": 12345, "cart_id": 5678 }
  ```
- Order Service (заказ отменён):
  ```json
  { "event_type": "order_cancelled", "user_id": 12345, "order_id": 1111 }
  ```
- Marketing Service (рассылка):
  ```json
  { "event_type": "promo_campaign", "segment": "all_users" }
  ```

**3. Message Broker** — Kafka (высокие нагрузки) или RabbitMQ (умеренные). Обеспечивает асинхронность и сохранность событий при падении сервисов.

**4. Notification Service** — читает события из брокера, определяет шаблон сообщения, получает push-токены пользователя, формирует уведомление и отправляет его через провайдера:

```json
{
  "title": "Вы забыли товары в корзине",
  "body": "Ваш заказ ожидает оформления уже 24 часа"
}
```

**5. Firebase/APNs** — доставляют push-уведомление на устройство пользователя.
